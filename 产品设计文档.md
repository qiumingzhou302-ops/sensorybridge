# SensoryBridge AI 产品设计文档（PRD）

> **项目名称**：听见画面·看见声音 —— AI 残障感官代偿创作工具包
> **英文名称**：SensoryBridge AI
> **文档版本**：v1.0
> **更新日期**：2026-06-16
> **产品定位**：国内首款面向残障人群的多模态感官代偿 AI 创作工具

## 一、项目分析

### 1.1 市场背景

- **人群规模**：国内残障人士约 8500 万，其中视障 1700 万、听障 2700 万、肢体障碍 2400 万
- **政策驱动**：国家《促进残疾人就业三年行动方案》明确鼓励残疾人居家灵活就业
- **技术契机**：AIGC 浪潮下，多模态大模型（文生图、语音识别、图像理解）已具备生产级能力
- **市场空白**：现有无障碍工具仅停留在信息浏览层面（读屏、实时字幕），数字创作领域完全空白

### 1.2 核心问题

| 障碍类型 | 创作痛点 | 现有方案缺陷 |
|---------|---------|------------|
| 视障 | 生成图片无法查看，创作全程依赖明眼人 | 读屏软件只能识别文字，无法解析画面 |
| 听障 | 剪辑视频听不到音频节奏，无法卡点对齐字幕 | 无音频可视化工具，字幕匹配度差 |
| 肢障 | 手部无力/震颤，无法完成拖拽、点击操作 | 现有软件交互依赖精细手部操作 |

### 1.3 解决思路

以 **多模态感官代偿** 为核心：

- **用听觉替代视觉**：AI 图像理解 + 语音播报，让视障用户"听见"画面
- **用可视化替代音频**：音频波形 + 节奏图形化，让听障用户"看见"声音
- **用语音替代手部操作**：全语音指令驱动，让肢障用户"说"出创作

### 1.4 竞品分析

| 竞品 | 定位 | 残障支持 | 差距 |
|-----|------|---------|------|
| Midjourney | 专业 AI 绘图 | 无 | 无无障碍设计 |
| 剪映 | 短视频剪辑 | 仅基础字幕 | 无音频可视化 |
| NVDA 读屏 | 网页浏览 | 仅文字读取 | 无法解析图像 |
| Be My Eyes | 视障协助 | 人工协助 | 非创作工具，依赖志愿者 |

**结论**：市场无直接竞品，SensoryBridge 填补"残障专属 AI 创作工具"赛道空白。

- **当前困境**：用 AI 生成图片后无法查看，需要朋友帮忙描述画面，修改全靠转述
- **核心诉求**：独立完成"生成-查看-修改"全流程，不依赖他人

- **当前困境**：听不到背景音乐节奏，剪辑全靠猜，字幕对不齐

#### 画像 D：特教老师 - 李老师

## 四、核心功能详细设计

### 4.1 视障创作模块

#### 4.1.1 语音文生图

**功能描述**：视障用户通过语音口述创意，AI 自动生成图片。

**交互流程**：
```
用户长按语音键 → 说出创意描述
    ↓
语音识别转文字 → 确认描述（语音播报）
    ↓
用户确认 → AI 生成图片（4张候选）
    ↓
语音播报："已为您生成4张图片，请选择"
    ↓
用户语音选择"第1张" → 确认使用
```

**关键设计**：
- 语音输入时长：单次最长 60 秒
- 支持中英文混合输入
- 生成过程语音反馈："正在为您创作，请稍候..."
- 生成失败语音提示并引导重试

**界面设计**：
- 大字体高对比度界面
- 语音按钮占据屏幕中央 60% 区域
- 实时语音波形可视化（给明眼人协助者参考）
- 生成结果以网格展示，每张图片支持语音描述

#### 4.1.2 听觉画面解析

**功能描述**：AI 将图片内容转化为详细的语音描述，让视障用户"听见"画面。

**解析维度**：

| 维度 | 描述内容 | 示例 |
|-----|---------|------|
| 整体构图 | 画面布局、主体位置 | "画面中央是一位少女，背景是夕阳下的海边" |
| 色彩光影 | 主色调、明暗对比 | "整体暖色调，金色阳光从右侧照射" |
| 人物细节 | 表情、动作、服饰 | "少女微笑着，长发随风飘动，身穿白色连衣裙" |
| 情感氛围 | 画面情绪感受 | "画面传达出温暖、宁静的氛围" |
| 细节元素 | 装饰、道具、文字 | "手中拿着一束向日葵，画面右下角有签名" |

**交互设计**：
- 点击图片 → 自动播报完整描述
- 支持分段播报："请描述构图" / "请描述色彩"
- 播报速度可调（0.5x - 2x）
- 支持语音打断："停" / "重复"

#### 4.1.3 语音指令修改

**功能描述**：用户通过语音指令修改已生成的图片。

**指令系统**：

| 指令类型 | 示例指令 | 效果 |
|---------|---------|------|
| 局部修改 | "把背景换成森林" | 替换背景 |
| 人物调整 | "让人物微笑" | 修改表情 |
| 色彩调整 | "色调更暖一些" | 调整色调 |
| 风格转换 | "改成水彩画风格" | 转换风格 |
| 构图调整 | "人物往左移一点" | 调整位置 |
| 重新生成 | "重新画一张" | 重新生成 |

**交互流程**：
```
用户："把背景换成森林"
    ↓
AI 确认指令："好的，正在将背景更换为森林"
    ↓
AI 修改图片 → 生成新版本
    ↓
语音播报新画面描述："已修改完成，现在背景是茂密的森林..."
    ↓
用户确认或继续修改
```

### 4.2 听障创作模块

#### 4.2.1 音频波形可视化

**功能描述**：将音频转化为直观的波形图和节奏图形，让听障用户"看见"声音。

**可视化形式**：

1. **波形图**：传统音频波形，显示音量变化
2. **频谱图**：彩色频谱显示，不同频率用不同颜色
3. **节奏脉冲**：节拍点用闪烁圆点标记
4. **情绪色带**：根据音频情绪（欢快/悲伤/激昂）显示对应颜色色带

**交互设计**：
- 波形图支持缩放（语音或手势）
- 节拍点自动检测并高亮标记
- 支持点击波形图任意位置跳转
- 字幕与波形图时间轴对齐显示

#### 4.2.2 智能字幕对齐

**功能描述**：AI 自动识别音频内容并生成字幕，与波形图精准对齐。

**功能点**：
- 自动语音识别生成字幕
- 字幕时间戳与波形图同步
- 字幕编辑支持文本输入（无需听音频）
- 支持多语言字幕生成

### 4.3 肢障创作模块

#### 4.3.1 全语音交互系统

**功能描述**：所有操作均可通过语音指令完成，无需精细手部操作。

**语音指令映射**：

| 操作类别 | 语音指令 | 对应操作 |
|---------|---------|---------|
| 文件操作 | "新建" / "保存" / "导出" | 文件管理 |
| 绘图操作 | "画一个圆" / "加文字" | 基础绘图 |
| 图层操作 | "新建图层" / "删除图层" | 图层管理 |
| 选择操作 | "选择左上角的图片" | 元素选择 |
| 调整操作 | "放大" / "缩小" / "左移" | 尺寸位置 |
| 滤镜操作 | "加复古滤镜" | 滤镜应用 |

**交互设计**：
- 常驻语音监听模式（可开关）
- 语音指令确认机制（重要操作需二次确认）
- 语音指令学习引导
- 支持自定义语音快捷指令

#### 4.3.2 简化操作界面

**设计原则**：
- 大按钮（最小 60x60px）
- 简化交互（单击替代拖拽）
- 语音优先（所有按钮均有语音替代方案）
- 键盘快捷键支持

### 4.4 通用功能模块

#### 4.4.1 无障碍设置中心

**功能描述**：用户可根据自身情况自定义无障碍设置，所有设置项均提供语音、键盘、点击三种操作方式，设置变更即时生效并语音播报确认。

- 设置中心本身必须完全无障碍（作为产品的"第一道门"）

- 设置持久化存储，跨设备同步

### 6.2 WCAG 2.1 四大原则落地设计

#### 6.2.1 原则一：可感知性（Perceivable）

|-----|------|-------------|---------|

| **1.2 时基媒体** | 为音频和视频提供替代 | 所有语音播报同步显示文字字幕；视频教程提供字幕+手语视频；音频内容提供文字摘要 | A/AA/AAA |

|-----|---------|---------|
| 页面加载 | 焦点移至 `<main>` 的 `<h1>` | `h1.focus()` |
| 路由切换 | 焦点移至新页面标题，播报页面名 | `useEffect` + `focus()` |
| 模态框打开 | 焦点移至模态框标题，Tab 循环在框内 | `aria-modal="true"` + focus trap |
| 模态框关闭 | 焦点返回触发元素 | 保存 `document.activeElement` 引用 |
| 图片生成完成 | 焦点移至结果区，播报"生成完成" | `aria-live` + `focus()` |
| Toast 出现 | 不抢焦点，通过 `aria-live` 播报 | `role="status"` |
| 下拉菜单 | 焦点移至菜单第一项，支持方向键导航 | `aria-expanded` + 键盘事件 |

### 11.2 技术选型说明

> **核心原则**：比赛项目优先，全 Serverless 架构，零服务器运维，免费额度覆盖比赛全周期。AI 能力调用国内云厂商 API，语音能力优先用浏览器原生 API（零成本）。

4. **Supabase 内置认证**：无需自建用户系统，直接用 Supabase Auth，省去后端开发量。
